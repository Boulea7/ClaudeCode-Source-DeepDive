# Why Some Developers Switched To Claude Code

This page keeps only a few repeated themes from public discussion. It is background, not proof of source-level conclusions.

## Common Reasons

- stronger end-to-end autonomy
- better fit for large repositories and long chains of work
- outputs that look closer to reviewable work products
- a terminal workflow that stays close to shell, git, and repo state
- plan, memory, team, MCP, and skills inside one runtime

## Reverse Reasons Matter Too

Some users still prefer Cursor, Windsurf, Aider, or Codex CLI for IDE interaction, model flexibility, lighter workflows, or open customization.
