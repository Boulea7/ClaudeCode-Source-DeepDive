# 为什么有人会从别的工具转向 Claude Code

这里不展开写社区情绪，只保留反复出现的几类原因。

## 常见原因

- 更强的端到端自治能力
- 更适合大代码库、多文件修改、长链路任务
- 输出更像“可审阅的工作结果”，不是只给建议
- 终端工作流更自然，和 shell / git / repo 更贴近
- `Plan Mode`、memory、team、MCP、skills 放在同一套运行时里

## 同样重要的反向原因

也有人继续留在 Cursor、Windsurf、Aider 或 Codex CLI。原因通常是：

- 更喜欢 IDE 交互
- 更需要多模型灵活性
- 更看重轻量、开源、易定制
