# 为什么有人会从别的工具转向 Claude Code

这里保留反复出现的几类原因。

这页的作用只是帮助读者理解“为什么大家会把它和别的工具放在一起讨论”，不是替源码结论做背书。

## 关键点

- 讨论焦点经常落在自治能力、长链路任务和终端工作流
- 也有人继续偏好 IDE、轻量路线或更高的可定制性
- 这些讨论能补背景，不能替代源码章节

## 常见原因

- 更强的端到端自治能力
- 更适合大代码库、多文件修改、长链路任务
- 输出更接近“可审阅的工作结果”
- 终端工作流更自然，和 shell / git / repo 更贴近
- `Plan Mode`、memory、team、MCP、skills 放在同一套运行时里

## 同样重要的反向原因

也有人继续留在 Cursor、Windsurf、Aider 或 Codex CLI。原因通常是：

- 更喜欢 IDE 交互
- 更需要多模型灵活性
- 更看重轻量、开源、易定制
