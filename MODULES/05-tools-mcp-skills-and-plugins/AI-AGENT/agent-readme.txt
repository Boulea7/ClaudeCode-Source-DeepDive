Module 05 explains the extensibility stack.

Focus on:
- services/mcp as the runtime backend
- tools vs resources as different object models
- skills as reusable workflow assets
- `getSkillToolCommands()` as listing, but `SkillTool -> getAllCommands()` as the wider execution set
- plugins as distribution and packaging surfaces
- builtin plugin wiring exists in `main.tsx`, but the current builtin plugin registry is still empty
