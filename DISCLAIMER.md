# Disclaimer

## 目的

本仓库仅用于：

- 源码研究
- 架构拆解
- 开发者文档整理
- feature gate / hidden branch 的证据边界梳理

它不是：

- 官方文档
- 官方发布说明
- 内部仓库镜像
- 功能可用性公告

## 材料来源

本仓库中的源码相关结论，只允许来自以下两处：

- `https://github.com/ChinaSiro/claude-code-sourcemap`
- 本地镜像：`_upstream/claude-code-sourcemap/`

如果某个判断不能从这两处直接复核，本仓库不会把它写成源码事实。

## 本仓库包含什么

本仓库主要包含：

- 架构总览
- 模块级源码走读
- prompt 装配机制说明
- MCP / skills / plugins / permissions / remote 等运行链拆解
- feature flag / hidden branch 的代码线索整理
- 面向 AI 的辅助索引材料

这些内容的重点是：

- 解释运行逻辑
- 标注关键源码路径
- 区分“已确认事实”和“仍待确认”

## 本仓库不包含什么

本仓库不包含：

- Claude Code 的完整还原源码
- 大段 raw prompt
- 无源码支撑的产品结论
- 把客户端兼容逻辑写成服务端事实的说明
- 把 gated branch 写成公开发布能力的公告

## 证据边界

如果当前公开镜像证据不足，文档只会写成以下几类表述：

- 有代码线索
- 当前镜像未完全坐实
- 不能过度下结论
- 不能确认公开发布状态

特别是下面这些名字，即使在代码里出现，也不应直接当成公开产品能力：

- `KAIROS`
- `PROACTIVE`
- `Buddy`
- `voice`
- `TEAMMEM`
- `CHICAGO_MCP`
- `remote isolation`
- `bridge`

## 与官方关系

本仓库是非官方研究仓库，与 Anthropic 无隶属关系，也不代表任何官方内部仓库结构、发布计划或产品口径。

这里对源码的整理与评论，属于研究者对公开镜像的阅读结果，不应被理解为官方确认。

## 知识产权与引用边界

原始 Claude Code 源码及相关权利不属于本仓库。

本仓库主要提供的是：

- 自行撰写的分析
- 结构化整理
- 少量用于说明问题的源码路径和结论

如果你继续引用这里的内容，建议同时保留下面三条：

1. 这是非官方研究文档。
2. 源码事实以公开镜像为边界。
3. gated branch 不等于正式公开功能。

## 文档写作原则

本仓库在写作时遵循以下原则：

- 先讲“是什么”，再讲“怎么实现”
- 每个重要结论尽量挂源码路径
- `PROMPTS/` 只解释装配机制，不贴大段 raw prompt
- `FEATURE-FLAGS/` 只写代码中的 gate / hidden branch，不写成发布说明
- 对证据不足的点，宁可写窄，不写满
